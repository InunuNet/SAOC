# Golden Spec â€” `scripts/refresh-llms.ts`

Standalone tsx script that queries Sanity via GROQ and writes `public/llms-full.txt`.
Run via `pnpm refresh-llms`. NO Alembic, NO HTTP crawling, NO `next-sanity`.

## Client setup (standalone `@sanity/client`)

```ts
import { createClient } from '@sanity/client';

const client = createClient({
  projectId: process.env.NEXT_PUBLIC_SANITY_PROJECT_ID!,
  dataset: process.env.NEXT_PUBLIC_SANITY_DATASET ?? 'production',
  apiVersion: '2024-10-01',
  useCdn: false,
  token: process.env.SANITY_API_TOKEN,
});
```

- Use the standalone `@sanity/client` `createClient` â€” NOT `next-sanity`, NOT the app's shared client.
- Queries are written **inline in the script** â€” do NOT import from `sanity/queries.ts`. The script must be self-contained.

## Sections

Emit sections in this order. Each section has a `## ` heading. Queries are inline GROQ strings.

| Section | GROQ | Format |
|---------|------|--------|
| `## About SAOC` | `*[_type == "aboutPage"][0]{ title, pillars, boardIntroText }` and `*[_type == "homePage"][0]{ missionText }` | `missionText` paragraph, then `pillars` as a bullet list |
| `## Affiliated Societies` | `*[_type == "society"] \| order(name asc){ name, province, region, meets, description }` | one line per society: `- **Name** (Province) â€” meets: ...; description` |
| `## Judging` | `*[_type == "judgingPage"][0]{ intro, howItWorks, stats }` | prose: `intro` text, then `howItWorks` steps |
| `## Events` | `*[_type == "societyEvent" && date >= now()] \| order(date asc)[0..9]{ title, date, kind, description, venue }` | list of upcoming events (date + title + venue); if empty: `No upcoming events scheduled.` |
| `## National Show` | `*[_type == "nationalShow"][0]{ title, showDate, location }` and `*[_type == "show" && status == "past"] \| order(year desc)[0..4]{ title, year, location, awards }` | upcoming show details, then past shows list |
| `## Contact` | `*[_type == "contactPage"][0]{ directContacts[]{ name, role, email } }` | contact list: `- **Name** (Role): email` |

## Error handling

- If `client.fetch()` returns `null` (or empty/undefined) for a query, write `_Content unavailable â€” Sanity not configured._` under that section's heading.
- Do NOT throw on missing content. The script must always produce a valid file.
- Log diagnostics to `process.stderr` (e.g. `process.stderr.write(...)`), never `console.log` to stdout.
