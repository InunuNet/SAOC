# Golden: scripts/refresh-llms.ts — Structure Spec

This is a structured specification, **not** code. The dev agent implements
`scripts/refresh-llms.ts` to match this shape. The script regenerates
`public/llms-full.txt` by crawling each site page through the Alembic proxy
(which strips each page to clean Markdown) and concatenating the output.

Only `llms-full.txt` is regenerated. `public/llms.txt` (curated index) stays
hand-authored and is never touched by this script.

## Imports (Node built-ins only — no new deps)

- `execSync` from `node:child_process` — run the Alembic `curl` per page
- `writeFileSync` from `node:fs` — write the assembled output
- `join` from `node:path` — build the absolute output path

`tsx` is already available (used by the `seed` script); do NOT add it as a dependency.

## Constants

- `PAGES`: an array of `{ path, title }` objects covering every public page, in order:
  | path            | title          |
  |-----------------|----------------|
  | `/`             | Home           |
  | `/about`        | About          |
  | `/societies`    | Societies      |
  | `/judging`      | Judging        |
  | `/events`       | Events         |
  | `/national-show`| National Show  |
  | `/contact`      | Contact        |
- `BASE_URL`: `process.env.REFRESH_BASE_URL ?? 'http://localhost:3001'`
- `ALEMBIC`: `http://localhost:7077` (the distilling proxy prefix)
- `OUTPUT_PATH`: `public/llms-full.txt` resolved relative to the project root

## Alembic crawl pattern (per page)

For each entry in `PAGES`:
1. Build the target URL: `${ALEMBIC}/${BASE_URL}${path}`
   (Alembic takes the full downstream URL as its path, e.g.
   `http://localhost:7077/http://localhost:3001/about`)
2. Fetch Markdown: `execSync(\`curl -s ${ALEMBIC}/${BASE_URL}${path}\`, { encoding: 'utf8' })`
3. Prepend an H2 header before the page body: `## ${title}\n\n`
4. Collect each section into an array.

Log progress to **stderr** (`process.stderr.write(...)` or `console.error(...)`),
never stdout — stdout stays clean. Example per-page log: `→ fetching /about`.

## Output assembly (in order)

1. **Org header block** (top of file):
   ```
   # South African Orchid Council — Full Site Content

   Source: SAOC website (saoc.co.za) — auto-generated by scripts/refresh-llms.ts
   Last updated: <ISO timestamp>
   ```
   The timestamp is `new Date().toISOString()`.
2. **Each page section** — `## <Title>` header followed by the page's Markdown,
   sections separated by a blank line, in `PAGES` order.
3. **WOSA scope-boundary footer** (bottom of file):
   ```
   ---
   ## Scope Boundary
   SAOC covers orchids in cultivation: growing, showing, hybridising, judging, community.
   Wild orchid conservation and identification → WOSA (Wild Orchids of Southern Africa): wosa.org.za
   ```

Join header + sections + footer with blank lines, then `writeFileSync(OUTPUT_PATH, content, 'utf8')`.

## Behavioural notes

- The script must exit 0 on success. It depends on two running services: the
  Next dev server on `:3001` (or `REFRESH_BASE_URL`) and the Alembic proxy on `:7077`.
- Fail fast: if a `curl` returns empty for a page, log a warning to stderr.
  (Do not silently write an empty section — at minimum keep the `## Title` header.)
- No secrets, no hardcoded credentials, no `console.log` to stdout.
